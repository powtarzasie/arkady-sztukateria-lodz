Jak powstała ta strona

Landing page „Sztukateria Łódź — ARKADY”: technologie, workflow assetów generatywnych, warstwa SEO i instrukcja uruchomienia.

← Wróć na stronę główną

1. Koncepcja i architektura

Strona została zaprojektowana i zbudowana w całości przez Claude (model Fable 5, Anthropic) w Claude Code. Kierunek artystyczny — „Katalog światła” — wywodzi się z realiów firmy: ARKADY posiada jedną z największych w Europie kolekcji ponadstuletnich katalogów detalu sztukatorskiego, więc strona jest złożona jak żywa tablica z takiego katalogu (numeracja „Tab. I–VII”, ramki grawerskie, adnotacje monospace), a separatorami sekcji są rysowane na scrollu przekroje profili ciągnionych.

  • Stack: czysty, statyczny HTML5 + CSS + vanilla JS (bez frameworków). Cała treść jest server-side w plikach HTML — pełna indeksowalność bez wykonywania JavaScriptu.
  • Serwer: minimalny serwer Node.js (server.js) z gzip, poprawnymi nagłówkami cache/MIME, czystą trasą /guide i prawdziwą obsługą 404 (status 404 + 404.html).
  • Animacje jako progressive enhancement: JS dodaje klasę js na <html>; dopiero wtedy CSS ukrywa elementy do scroll-revealu. Bez JS (i dla Googlebota) wszystko jest widoczne od razu. Całość respektuje prefers-reduced-motion.

3. Assety generatywne

Obrazy — GPT Image 2 (Higgsfield)

13 fotorealistycznych obrazów wygenerowano modelem GPT Image 2 przez Higgsfield CLI w rozdzielczości 2K. Spójność serii zapewnia wspólny sufiks stylistyczny w promptach: „photorealistic fine-art architectural photography, warm chalk-white gypsum, raking light, subtle film grain, no people, no text”. Nazwy plików są opisowe i polskie (SEO), np. gzymsy-sufitowe-fasety-ciagnione-lodz-800.avif.

Animacja — Seedance 2.0 (Higgsfield)

Obraz hero został ożywiony modelem Seedance 2.0 (image-to-video, 8 s, 1080p): statyczna kamera, smuga słońca wędrująca po reliefie, drobiny kurzu. Wideo skompresowano przez ffmpeg (H.264, CRF 28, 1600 px, bez audio) z 5,3 MB do ~0,55 MB i doładowywane jest wyłącznie: po pełnym załadowaniu strony, na ekranach ≥ 820 px, gdy użytkownik nie ma prefers-reduced-motion ani Save-Data. LCP to zawsze statyczny obraz AVIF.

Pipeline optymalizacji

node tools/build-images.mjs   # sharp: AVIF (q52) + WebP (q74) + JPG (q78)
                              # w 3–4 szerokościach na obraz + og-image
node tools/build-favicons.mjs # favicon.svg -> PNG 180/192/32 + favicon.ico

4. Warstwa SEO

  • Jedno <h1>: „Sztukateria Łódź — gips, który rzeźbi światło”; hierarchia H2/H3 odwzorowuje usługi (gzymsy sufitowe, rozety sufitowe, listwy ozdobne, renowacja sztukaterii…).
  • <title> 54 znaki, meta description ~150 znaków, canonical, OG + Twitter Card z dedykowanym obrazem 1200×630, lang="pl", favicon set.
  • Semantyczne landmarki: header / nav / main / section / footer; nawigacja kotwiczna do sekcji; opisowe polskie alt na każdym obrazie.
  • JSON-LD (@graph): WebSite, WebPage, HomeAndConstructionBusiness (NAP: ul. Tuwima 13, 90-010 Łódź, tel. +48 605 068 288, geo 51.7668/19.4611, godziny otwarcia, obszar działania, sameAs) z hasOfferCatalog (7 usług jako Service) oraz FAQPage (8 pytań zgodnych 1:1 z widoczną sekcją FAQ). BreadcrumbList pominięto świadomie — strona jest jednopoziomowa, nie ma ścieżki okruszków.
  • Local SEO: treść osadzona w realiach Łodzi (Piotrkowska 243, kamienice, pałace fabrykanckie), sekcja mapy z NAP, spójny NAP w stopce.
  • Pliki techniczne: robots.txt (z sitemapą, /guide wyłączony z indeksacji), sitemap.xml z rozszerzeniem image, strona 404 zwracająca prawdziwy status 404.
  • Uwaga wdrożeniowa: canonical i adresy w sitemapie wskazują na https://sztukateria.arkady.co/. Przy publikacji pod inną domeną należy podmienić domenę w index.html (canonical, OG, JSON-LD), sitemap.xml i robots.txt. Godziny otwarcia (pn.–pt. 8–16) przyjęto jako typowe dla pracowni — zweryfikuj przed publikacją.

5. Wydajność (Core Web Vitals)

  • LCP: hero jako <picture> AVIF/WebP/JPG z srcset/sizes, fetchpriority="high" i <link rel="preload"> z imagesrcset.
  • CLS ≈ 0: jawne width/height na każdym medium.
  • Obrazy poniżej foldu: loading="lazy" + decoding="async".
  • JS: ~3 kB, defer, zero bibliotek; CSS: jeden plik, gzip po stronie serwera; fonty preloadowane (Marcellus).
  • Wideo hero nie blokuje niczego — dołącza po window.load i tylko na desktopie.

6. Jak uruchomić lokalnie

# wymagany Node.js 18+
cd arkady-sztukateria-lodz
node server.js
# → http://localhost:4321        (strona)
# → http://localhost:4321/guide  (ten przewodnik)

Serwer nie ma żadnych zależności produkcyjnych (czysty Node). Zależności deweloperskie (sharp, ffmpeg-static) potrzebne są wyłącznie do regeneracji assetów: npm install, potem skrypty z katalogu tools/.
